# RQM — автоматическая навигация для Telegram-канала переводов

Система автоматической навигации по приватному каналу переводческой команды
**RQM**. Автор постит главы новелл/манги (и арты/мемы/заметки/анонсы), бот следит
за каналом и поддерживает навигацию в актуальном состоянии **без ручного труда**.

Навигация существует в двух видах поверх одной БД:

1. **Telegraph-страницы** — статичная витрина (главная закрепляется в канале):
   главная → проекты → главы по аркам, разделы (Арты/Мемы/Уголок/Анонсы).
2. **Бот** — мгновенный поиск для всех + полное управление для админов прямо в
   переписке (без отдельного веб-приложения).

```
Канал (автор постит)
      │
   БОТ (watcher, polling) ── хэштег-центричный разбор
      │
   SQLite  ◄── единственный источник истины
   /        \
Telegraph    Бот: поиск (все) + CRUD-админка (админы)
страницы
(закреп)
```

> Telegram Mini App из проекта **исключён** намеренно — весь UX в боте, он
> надёжно работает в РФ и не требует отдельного хостинга фронтенда.

## Возможности

**Для всех (поиск в боте):**
- Пришлите `304`, `глава 304`, `покровитель 305`, `турнир`, `башня` — бот
  мгновенно выдаёт совпадения со ссылками «📖 Читать» (Telegraph) и «💬 Пост».
  Поиск терпим к опечаткам в названии. Лимит — 10 запросов в минуту на
  пользователя (админы без лимита).

**Для админов (владелец канала, его администраторы, либо `OWNER_USER_IDS`):**
- **Проекты:** имя, эмодзи, ссылки RanobeLib/MangaLib/Senkuro/Boosty, порядок,
  скрыть/показать, хэштеги проекта, **вид** (группа); главы и арки —
  редактировать/удалять/переименовывать/объединять/разбивать.
- **Виды произведений** (Манга/Манхва/Новеллы): создать/переименовать/эмодзи/
  удалить, хэштеги вида; произведения раскладываются по видам.
- **Разделы:** создать/переименовать/эмодзи/порядок/удалить, записи и хэштеги.
- **Хэштеги:** привязать к проекту/виду/разделу, удалить.
- **Операции** (кнопки в панели «🛠 Админка»): 📊 Статус, 🔗 Ссылки на
  Telegraph-страницы, ♻️ Пересобрать навигацию.

Вся админка — через кнопку «🛠 Админка» на нижней клавиатуре (отдельных команд
нет, чтобы не дублировать кнопки).

**Цитирование главы** командой `/quote` (для всех — в личке и в группах, куда
добавлен бот): `/quote <тайтл> глава N абзацы A-B` или `… от "фраза" до "фраза"`;
в личке без диапазона бот показывает нумерованные абзацы для выбора. В личке также
кнопка «📄 Цитировать главу» в карточке произведения. Текст берётся с Telegraph и
кэшируется; фрагмент, не помещающийся в одно сообщение (по символам и строкам),
отклоняется с просьбой сузить диапазон.

Доступ к админке определяется **динамически**: админом считается создатель
канала, любой его администратор, либо id из `OWNER_USER_IDS`.

## Архитектура (хэштег-центричная)

Проект/категория поста определяется **только хэштегом**:
- `#покровитель`, `#гений`, … → проект: разбираем тело, достаём все главы.
- `#арты`, `#мемы`, `#заметки_переводчика`, `#анонсы` → категория: запись =
  первая строка поста + ссылка на сам пост (контент — картинка/текст в посте,
  Telegraph там нет).
- неизвестный тег → автосоздаётся раздел + уведомление админам.
- без тега → служебный пост, в навигацию не идёт.

**Telegraph = только главы.** Арты/мемы/заметки/анонсы хранятся как контент
самого Telegram-поста; навигация ведёт на пост.

## Структура проекта

```
app/
  config.py     конфиг из переменных окружения (+ мини-загрузчик .env)
  db.py         SQLite-слой (схема, миграции, CRUD, умный поиск)
  parser.py     парсер live-сообщений Telegram
  registry.py   реестр проектов/разделов/хэштегов + классификация
  seed.py       посев реестра (идемпотентно)
  telegraph.py  async-клиент Telegraph с ретраями
  render.py     сборка Telegraph-страниц (главная/проект/раздел, пагинация)
  rebuild.py    воркер очереди пересборки (дедуп по content-hash)
  pipeline.py   live-обработка постов (хэштег → проект/категория)
  bot.py        бот: watcher + поиск + цитаты (/quote) + CRUD-админка
  api.py        минимальный health-сервер (liveness для хостинга)
  main.py       entrypoint: бот + health-сервер + воркер + reconciler
scripts/        setup_telegraph, whoami
tests/          тесты (parser, render, rebuild, pipeline, quote, search, config)
```

## Локальный запуск

```bash
python -m pip install -r requirements.txt
cp .env.example .env          # заполнить BOT_TOKEN, CHANNEL_CHAT_ID и т.д.

python -m pytest -q                # тесты
python -m app.main                 # запустить всё
```

### Секреты (`.env`)
- **BOT_TOKEN** — у [@BotFather](https://t.me/BotFather). Бот должен быть
  **админом канала** (иначе не видит посты).
- **CHANNEL_CHAT_ID** — `python -m scripts.whoami`, затем пост в канал.
- **OWNER_USER_IDS** — необязательно (админы канала и так получают доступ); можно
  указать дополнительных управляющих, которые не являются админами канала.
- **TELEGRAPH_TOKEN** — `python -m scripts.setup_telegraph` (или создастся
  автоматически при первом запуске и сохранится в БД; в логи токен не выводится).
- **TELEGRAM_PROXY** — для РФ-сетей, если `api.telegram.org` недоступен напрямую
  (HTTP/SOCKS5). На зарубежном сервере не нужен.

## Деплой (бесплатно, стабильно в РФ, 24/7)

Бот ходит наружу к `api.telegram.org` (с зарубежного сервера это работает),
Telegraph-страницы открываются у всех. Mini App-домен не нужен.

### Oracle Cloud Always Free (рекомендуется)
Вечно бесплатная VM, свой диск под SQLite, реальный 24/7.
```bash
sudo apt install -y docker.io docker-compose-plugin
git clone <repo> rqm && cd rqm
cp .env.example .env && nano .env
docker compose up -d --build
docker compose logs -f
```

### Fly.io
```bash
fly launch --no-deploy
fly volume create rqm_data --size 1 --region fra
fly secrets set BOT_TOKEN=... OWNER_USER_IDS=... TELEGRAPH_TOKEN=... \
    CHANNEL_CHAT_ID=-100xxxxxxxxxx
fly deploy
```

> **Важно:** SQLite — на **persistent volume** (`/data`). Эфемерная ФС теряет
> данные при редеплое.

### После деплоя
1. В логах — `full rebuild complete`.
2. В боте: «🛠 Админка» → «🔗 Ссылки» → возьми ссылку на главную
   Telegraph-страницу → **закрепи её в канале**.

### Разработка → production
Разрабатывай и проверяй изменения в ветке `developer`. Когда версия готова:
`git checkout master && git merge developer && git push origin master`.
Push в `master` проверяется GitHub Actions job `Test`. Production-сервер раз в
минуту проверяет `origin/master` через systemd timer `rqm-auto-deploy.timer`; при
новом commit он подтягивает код, пересобирает Docker-контейнер, прогоняет
smoke-check и перезапускает бота. Если smoke-check после обновления падает,
серверный скрипт пытается откатиться на предыдущий commit.

Секреты, `.env` и SQLite-БД хранятся только на сервере. Ежедневный snapshot БД
сохраняется в `/data/backups/` и отправляется владельцам/админам канала в личку.
Проверить бэкап вручную: `python -m scripts.check_backup /path/to/backup.db`.

## Надёжность

- **Идемпотентность:** посты дедуп по `message_id`, главы по `(project, number)`.
- **Очередь пересборки** дебаунсит пачки постов; дедуп по content-hash → Telegraph
  не дёргается зря.
- **Периодическая само-пересборка** (`RECONCILE_INTERVAL_MIN`, по умолч. 30 мин):
  навигация не «уплывает» от БД. Учти: Bot API не читает историю канала и не видит
  удаления — это пере-публикация из БД, а не ре-скан старых постов.
- **БД навигации не засоряется пользовательскими запросами:** поиск, `/quote` и
  сообщения в группах не создают посты/главы/разделы; навигационные таблицы
  обновляются только из постов канала и действий админки.
- **Неизвестные хэштеги безопасны:** по умолчанию они создают conflict и
  уведомление админу, но не публикуются в навигацию, пока админ их не привяжет.
- **Безопасная публикация:** контент валидируется до `editPage`; пустое не
  перезатирает рабочую страницу.
- **Тяжёлые операции ограничены мягкими таймаутами:** rebuild/download/quote не
  висят бесконечно, но лимиты достаточно большие для медленной сети.
- **Ретраи Telegraph** + обработка `FLOOD_WAIT`. Стартовые ретраи бота на случай
  тормозов сети РФ.
- **Восстановление после рестарта** полностью из БД (persistent volume).
- Нераспознанное складывается в `conflicts`, а не теряется; статус — в «🛠 Админка → 📊 Статус».

## Зафиксированные решения

- Группировка глав — по **аркам**; сортировка по номеру.
- Длинные проекты Telegraph бьются на «Части» (главная проекта = индекс).
- Приоритет площадок: RanobeLib → MangaLib → Senkuro → Boosty (Senkuro добавлен —
  найден в истории).
- Стек: `python-telegram-bot` v21 (polling), `aiosqlite`, `aiohttp` (health).
- Mini App исключён — весь UX в боте.

## Тесты

```bash
python -m pytest -q
```
Покрытие: парсер (форматы, номера/арки/URL, UTF-16, голые ссылки), рендер
(валидность нод, размер, пагинация), пересборка (оркестрация, hash-skip),
live-пайплайн (хэштег→главы, неизвестный тег, категории, правки), цитаты,
умный поиск, конфиг (ссылки на канал).
