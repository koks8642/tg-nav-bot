# RQM — автоматическая навигация для Telegram-канала переводов

Система автоматической навигации по приватному каналу переводческой команды
**RQM**. Автор постит главы новелл (и арты/мемы/заметки), бот следит за каналом и
поддерживает навигацию в актуальном состоянии **без ручного труда**, в двух
представлениях поверх одной БД:

1. **Telegraph-страницы** — статичная витрина (главная закрепляется в канале).
2. **Telegram Mini App** — интерактив с умным поиском глав.

Оба представления читают одну SQLite-базу — единственный источник истины. Бот
пишет в неё, Telegraph-рендерер и Mini App API читают из неё → всегда
консистентны.

```
Канал (автор постит)
      │
   БОТ (watcher, polling) ── хэштег-центричный разбор
      │
   SQLite  ◄── единственный источник истины
   /     \
Telegraph   Mini App API (HTTP)
страницы    + фронтенд (поиск/навигация)
(закреп)    (кнопка в закрепе)
```

## Архитектура (хэштег-центричная)

**Live-режим.** Проект/категория поста определяется **только хэштегом** (один
хэштег на пост). Хэштег ищется в `hashtag_map`:

- `#покровитель`, `#гений`, … → **проект**: бот разбирает тело, достаёт все главы
  (1–8 ссылок), апсертит их.
- `#арты`, `#мемы`, `#заметки_переводчика`, `#анонсы` → **категория**: создаётся
  айтем со ссылкой.
- неизвестный хэштег → **автосоздаётся раздел** + уведомление владельцу.
- без хэштега → служебный пост, в навигацию не идёт.

**Бэкафилл истории (одноразовый, структурный).** У старой истории хэштегов нет,
поэтому проект определяется структурно — по заголовку `Новелла: "..."` /
префиксу `<Название>, Глава N` / слагу Telegraph, через реестр алиасов. Главы
извлекаются тем же парсером тела, что и в live. **Структурный матчинг
используется ТОЛЬКО в бэкафилле.**

**Ключевая находка по данным.** В истории есть 2 поста-агрегатора («Навигация по
тайтлу»), которые дублируют ссылки из анонсов. Дедуп по `(project, number)` с
приоритетом «релиз-пост > агрегатор» это решает; «Пост в канале» указывает на
реальный анонс, а не на агрегатор (который мы как раз заменяем).

## Структура проекта

```
app/
  config.py        конфиг из переменных окружения (+ мини-загрузчик .env)
  db.py            SQLite-слой (схема, миграции, CRUD, умный поиск)
  parser.py        парсер постов (экспорт HTML + live-сообщения) — ядро
  registry.py      реестр проектов/разделов/хэштегов + классификация постов
  seed.py          посев реестра в БД (идемпотентно)
  backfill.py      одноразовый структурный бэкафилл из экспорта
  telegraph.py     тонкий async-клиент Telegraph с ретраями
  render.py        сборка node-формата страниц (главная/проект/раздел, пагинация)
  rebuild.py       воркер очереди пересборки (дедуп по хэшу, безопасная публикация)
  pipeline.py      live-обработка постов (хэштег → проект/категория)
  bot.py           бот: watcher канала + админ-меню (PTB v21)
  webapp_auth.py   валидация подписи Telegram initData
  api.py           aiohttp: публичное чтение + защищённый admin-write + статика
  main.py          entrypoint: бот + HTTP + воркер в одном loop
webapp/            Mini App (index.html + app.js + styles.css), ванильный JS
scripts/           setup_telegraph, whoami, run_backfill, analyze_export
tests/             40 тестов (parser, backfill, render, rebuild, pipeline, api, auth)
```

## Локальный запуск

```bash
python -m pip install -r requirements.txt
cp .env.example .env          # заполнить BOT_TOKEN, OWNER_USER_IDS и т.д.

# 1) посмотреть, что распарсится из экспорта (без БД)
python -m scripts.analyze_export

# 2) прогнать бэкафилл (создаёт data/rqm.db)
python -m scripts.run_backfill

# 3) тесты
python -m pytest -q

# 4) запустить всё (бот + API + воркер)
python -m app.main
```

### Получить секреты

- **BOT_TOKEN** — у [@BotFather](https://t.me/BotFather). Бота сделать
  **админом канала** (право читать посты).
- **CHANNEL_CHAT_ID** — `-1003131929652` (для приватного канала
  `3131929652`). Проверить: `python -m scripts.whoami`, затем запостить в канал.
- **OWNER_USER_IDS** — ваш user_id: напишите боту `/id`.
- **TELEGRAPH_TOKEN** — `python -m scripts.setup_telegraph` (или бот создаст
  сам при первом запуске и сохранит в БД; для надёжности вынесите в env).
- **WEBAPP_URL** — HTTPS-домен, где раздаётся Mini App (даёт хостинг).

## Деплой (бесплатно, стабильно в РФ, 24/7)

Telegram Mini App открывается **внутри Telegram**, поэтому доступен из РФ, пока
работает сам Telegram. Серверу нужно лишь уметь ходить в `api.telegram.org`
(хостинг вне РФ это умеет) и иметь HTTPS-домен для Mini App.

### Вариант A — Oracle Cloud Always Free (рекомендуется: реально бесплатно 24/7)

Вечно бесплатная ARM-VM (до 4 ядер / 24 ГБ), свой диск под SQLite, полноценный
24/7. HTTPS даём через Caddy (авто-Let's Encrypt) или Cloudflare Tunnel.

```bash
# на VM (Ubuntu):
sudo apt update && sudo apt install -y docker.io docker-compose-plugin
git clone <repo> rqm && cd rqm
cp .env.example .env && nano .env      # секреты; WEBAPP_URL=https://ваш-домен
docker compose up -d --build
docker compose logs -f
```

HTTPS перед контейнером (нужен для Mini App), пример Caddy:

```
ваш-домен {
    reverse_proxy localhost:8080
}
```

> Нет домена? Бесплатный поддомен (DuckDNS) + Caddy, либо Cloudflare Tunnel
> (`cloudflared`) — он сам даёт HTTPS-URL и не требует открытых портов.

### Вариант B — Fly.io (проще, free allowance, persistent volume)

```bash
fly launch --no-deploy
fly volume create rqm_data --size 1 --region fra
fly secrets set BOT_TOKEN=... OWNER_USER_IDS=... TELEGRAPH_TOKEN=... \
    CHANNEL_CHAT_ID=-1003131929652 WEBAPP_URL=https://<app>.fly.dev
fly deploy
```

`fly.toml` уже настроен: persistent volume на `/data`, `auto_stop_machines=false`
(polling-бот должен жить всегда), HTTPS из коробки.

### Вариант C — Railway

Совместимо (Dockerfile + volume на `/data`), но платно (~$5/мес после триала).
Деплой из GitHub-репо, добавить volume на `/data`, задать переменные.

> **Важно:** SQLite обязана лежать на **persistent volume** (`/data`).
> Эфемерная ФС теряет данные при редеплое.

### Финальные шаги после деплоя

1. Дождаться в логах `full rebuild complete` — Telegraph-страницы созданы.
2. В админ-боте `/health` → проверить счётчики.
3. Закрепить главную Telegraph-страницу в канале (её URL — в БД
   `telegraph_pages` kind=`root`, или из лога).
4. В @BotFather → `/setmenubutton` → задать Mini App кнопку на `WEBAPP_URL`
   (или добавить `web_app`-кнопку в закреплённый пост).

## Админка (полный CRUD без правки кода)

Две точки входа, один защищённый write-API:

- **Mini App-админка** (основной удобный инструмент) — открывается владельцем
  (кнопка 🛠 или `?admin=1`). Формы: проекты (имя, эмодзи, ссылки
  RanobeLib/MangaLib/Senkuro/Boosty, порядок, скрытие), хэштеги (привязка
  `#тег → проект/категория`), очередь конфликтов, ручные действия (пересборка,
  бэкафилл).
- **Бот-меню** (быстрый доступ с телефона) — `/start` или `/menu`: health,
  конфликты (с кнопкой «решить»), список проектов/хэштегов, полная пересборка,
  `/backfill`, `/rebuild`.

Все write-операции проверяют подпись `initData` и whitelist `OWNER_USER_IDS` на
сервере (клиенту не доверяем). Каждая правка пишется в `audit_log`.

## Надёжность

- **Идемпотентность** везде: посты дедуп по `message_id`, главы по
  `(project, number)`, айтемы/ссылки по естественным ключам.
- **Очередь пересборки** дебаунсит пачки постов (воркер опрашивает каждые 3 c;
  дубликаты в очереди схлопываются).
- **Дедуп по content-hash**: страница не пересобирается в Telegraph, если её
  содержимое не изменилось → экономия rate-limit.
- **Безопасная публикация**: контент собирается и валидируется до `editPage`;
  пустой/битый контент никогда не перезатирает рабочую страницу.
- **Ретраи Telegraph** с экспоненциальным backoff + обработка `FLOOD_WAIT`.
- **Восстановление после рестарта** полностью из БД; никакого состояния только в
  памяти. Polling переподтягивает пропущенные апдейты (Telegram хранит ~24 ч).
- **Бэкап БД** перед массовыми операциями (бэкафилл).
- **Логи** в `event_log` + `/health`; нераспознанное складывается в `conflicts`,
  а не теряется.

## Зафиксированные решения (открытые вопросы из ТЗ)

- **Группировка глав — по аркам** (арки надёжно парсятся из меток/заголовков;
  явных томов в постах нет). Сортировка по номеру по возрастанию.
- **Пагинация Telegraph**: длинные проекты режутся на «Части» (главная проекта
  становится индексом со ссылками на части). Покровитель (191 глава) ≈ 77 КБ >
  лимита → автоматически бьётся на части; Гений умещается в одну страницу.
- **Приоритет площадок**: RanobeLib → MangaLib → Senkuro → Boosty.
- **Senkuro** добавлен как площадка (найден в истории, в ТЗ не упомянут).
- **Стек**: `python-telegram-bot` v21 (polling), `aiosqlite`, `aiohttp` (и
  Telegraph-клиент, и API, и статика — один процесс, минимум зависимостей).
