# Гайд: тестируем бота на своём тестовом канале (Windows)

Цель — за ~15 минут поднять бота локально на своём ПК и проверить всю логику на
тестовом Telegram-канале. Хостинг для этого **не нужен** (кроме опциональной
проверки Mini App — см. часть 8).

---

## 0. Что понадобится

- Python 3.11+ (у тебя есть)
- Аккаунт Telegram
- 10 минут

В корне проекта уже всё установлено. Если в новой среде — поставь зависимости:

```powershell
python -m pip install -r requirements.txt
```

---

## 1. Создай тестовый канал

1. Telegram → ✏️ (Новое сообщение) → **Создать канал**.
2. Название любое, например «RQM Test». Тип — **Частный** (как у боевого).
3. Готово. Запости туда что угодно для проверки.

---

## 2. Создай бота

1. Открой [@BotFather](https://t.me/BotFather) → `/newbot`.
2. Имя (любое) и username (должен заканчиваться на `bot`, например `rqm_test_bot`).
3. BotFather пришлёт **токен** вида `8123456789:AAH...` — скопируй, это `BOT_TOKEN`.

---

## 3. Сделай бота админом канала

1. Канал → ⚙️ → **Администраторы** → **Добавить администратора**.
2. Найди своего бота по username, добавь.
3. Права: достаточно дефолтных. Главное — бот должен быть админом, иначе он
   **не видит посты канала** (это требование Telegram).

---

## 4. Узнай chat_id канала и свой user_id

В `.env` пока впиши только токен (остальное узнаем сейчас):

```powershell
Copy-Item .env.example .env
notepad .env
```

Впиши `BOT_TOKEN=...` и сохрани. Запусти помощник:

```powershell
python -m scripts.whoami
```

Теперь:
- **Свой user_id:** напиши боту в личку любое сообщение (например `привет`).
  В консоли появится `user_id=123456789` — это твой `OWNER_USER_IDS`.
- **chat_id канала:** запости что-нибудь в тестовый канал. В консоли появится
  `chat_id=-100XXXXXXXXXX  -> CHANNEL_CHAT_ID=-100XXXXXXXXXX`.

Нажми `Ctrl+C`, чтобы остановить помощник.

> Если бот не реагирует на сообщения в личке — открой бота и нажми **Start**.

---

## 5. Заполни .env

Открой `.env` (`notepad .env`) и заполни:

```ini
BOT_TOKEN=8123456789:AAH...           # из шага 2
CHANNEL_CHAT_ID=-100XXXXXXXXXX        # из шага 4 (ТВОЙ тестовый канал!)
OWNER_USER_IDS=123456789             # из шага 4
# TELEGRAPH_TOKEN можно не трогать — бот создаст сам при первом запуске
DB_PATH=./data/rqm.db
EXPORT_HTML=./ChatExport/messages.html
```

Остальное оставь по умолчанию.

> ⚠️ `CHANNEL_CHAT_ID` обязательно поменяй на свой тестовый канал, а не оставляй
> боевой `-1003131929652` — иначе бот будет слушать не тот канал.

---

## 6. Запусти бота

```powershell
python -m app.main
```

Что произойдёт при первом старте:
- создастся БД `data/rqm.db`;
- посеется реестр (проекты Покровитель/Гений/… и хэштеги `#покровитель`,
  `#гений`, `#арты`, `#мемы`, `#заметки_переводчика`, `#анонсы`);
- создастся Telegraph-аккаунт (токен сохранится в БД);
- соберётся главная Telegraph-страница и пустые страницы разделов;
- бот начнёт слушать канал (увидишь `Bot polling started`).

Оставь это окно открытым. В личке боту отправь `/start` — откроется админ-меню.
Команда **`/links`** покажет ссылку на главную Telegraph-страницу и страницы
проектов.

---

## 7. Тест-посты (постим в тестовый КАНАЛ)

Скопируй и запости в канал **как обычный пост**. Между постами жди 3–5 секунд
(воркер пересобирает страницы). После каждого проверяй результат (см. «✓»).

> Ссылки `telegra.ph/...` в постах можно вставлять **голыми** — бот их понимает.
> Они ведут на несуществующие страницы (это нормально для теста логики). Хочешь
> кликабельные «Читать» — создай реальные страницы на telegra.ph (без регистрации,
> 10 секунд) и вставь их URL.

### Тест 1 — пачка глав проекта

```
🎁 Новые главы 🎁
#покровитель
Главы 305-307 «Финал»

https://telegra.ph/Stal-Pokrovitelem-Zlodeev-Glava-305-Final-06-03
https://telegra.ph/Stal-Pokrovitelem-Zlodeev-Glava-306-Final-06-03
https://telegra.ph/Stal-Pokrovitelem-Zlodeev-Glava-307-Final-06-03
```

**✓ Ожидаем:** в проекте «Стал Покровителем Злодеев» появились главы 305–307,
арка «Финал». Проверь: боту `/links` → открой страницу проекта → там новая арка.
Боту `/health` → счётчик глав вырос на 3.

### Тест 2 — одиночная глава другого проекта

```
#гений
Глава 58 «Возвращение»
https://telegra.ph/Oshibochno-Prinyali-Za-Velichajshego-Geniya-Glava-58-Vozvrashchenie-06-03
```

**✓ Ожидаем:** в проекте «Ошибочно Приняли За Величайшего Гения» глава 58, арка
«Возвращение».

### Тест 3 — категория (арт)

```
#арты
Обложка к новелле от художника
https://telegra.ph/Art-Cover-06-03
```

**✓ Ожидаем:** в разделе «Арты» появилась запись. `/links` → открой «Арты».

### Тест 4 — неизвестный хэштег (автораздел + уведомление)

```
#спойлеры
Большой спойлер по сюжету, осторожно!
```

**✓ Ожидаем:** бот **пришлёт тебе в личку** уведомление «🆕 Новый неизвестный
хэштег #спойлеры…». Автоматически создался раздел «#спойлеры». Боту `/start` →
«⚠️ Конфликты» → увидишь запись с кнопкой «✓ Решить».

### Тест 5 — редактирование (правка поста)

Отредактируй **пост из теста 1**: поменяй `«Финал»` на `«Финал, том 2»` или
добавь ещё одну строку-ссылку `...-Glava-308-Final-06-03`.

**✓ Ожидаем:** навигация обновилась (новая глава 308 / новое имя арки), дублей
нет — глава с тем же номером перезаписалась.

### Тест 6 — пост без хэштега (игнор)

```
Всем привет! Сегодня глав не будет, отдыхаю 🌚
```

**✓ Ожидаем:** ничего не добавилось в навигацию (служебный пост). `/health` —
счётчики глав/айтемов не изменились.

### Тест 7 — дедуп/идемпотентность

Запости **ещё раз тот же текст, что в тесте 1** (новый пост).

**✓ Ожидаем:** главы 305–307 **не задублировались** — в проекте всё те же 3
главы (ссылка обновилась на новый пост).

## 8. (Опционально) Проверка Mini App

Mini App требует HTTPS. Локально это даёт бесплатный туннель **cloudflared**:

```powershell
winget install --id Cloudflare.cloudflared
# в отдельном окне, пока app.main запущен:
cloudflared tunnel --url http://localhost:8080
```

Cloudflared выдаст HTTPS-URL вида `https://random-words.trycloudflare.com`.

1. Впиши его в `.env`: `WEBAPP_URL=https://random-words.trycloudflare.com`
2. Перезапусти `python -m app.main`.
3. В [@BotFather](https://t.me/BotFather): `/setmenubutton` → выбери бота →
   текст кнопки «Навигация» → URL = твой `WEBAPP_URL`.
4. Открой бота в Telegram → нажми кнопку меню → откроется Mini App: поиск,
   проекты, разделы, лента. Как владелец увидишь кнопку 🛠 (админка).

Проверь умный поиск: введи `покровитель 305`, `305`, `финал` — должны находиться
твои тестовые главы.

> Адрес trycloudflare меняется при каждом запуске — это только для теста. На
> проде будет постоянный домен (см. README → Деплой).

## Шпаргалка по командам бота (в личке)

| Команда | Что делает |
|---|---|
| `/start`, `/menu` | админ-меню (health, конфликты, проекты, хэштеги, пересборка) |
| `/links` | ссылки на главную и все Telegraph-страницы |
| `/health` | счётчики, очередь, последние ошибки |
| `/id` | твой user_id и chat_id |
| `/rebuild` | пересобрать все страницы |
| `/backfill` | (для боевого канала) импорт истории из экспорта |

---

## Если что-то не так

- **Бот не реагирует на посты канала** → бот не админ канала, либо в `.env`
  чужой `CHANNEL_CHAT_ID`. Проверь шаг 3 и 5.
- **«BOT_TOKEN is not set»** → не заполнен `.env` или запуск не из корня проекта.
- **Главы не извлекаются** → в ссылке нет паттерна `-Glava-<номер>-`. Используй
  URL как в тест-постах (важна часть `Glava-305`).
- **Telegraph-страница не создаётся** → посмотри логи в окне `app.main` и боту
  `/health` (последние ошибки). Часто помогает `/rebuild`.
- **Сбросить всё и начать заново** → останови бота, удали `data/rqm.db*`, запусти
  снова.
